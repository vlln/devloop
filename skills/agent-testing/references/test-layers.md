# 测试层级定义

从最内层向外逐层叠加。每层都是合并闸门，不可跳过。

## 按项目类型确定所需层级

| 项目类型 | 所需层级 | 说明 |
|----------|---------|------|
| 纯 CLI / 脚本工具 | ① 单元 → ② 集成 | 集成测试使用退出码、stdout、文件断言 |
| 后端 API 服务 | ① 单元 → ② 集成 → ③ 接口契约 → ④ 接口编排 | 接口编排覆盖多接口时序联动 |
| 全栈 Web 应用 | ① 单元 → ② 集成 → ③ 接口契约 → ④ 接口编排 → ⑤ E2E → ⑥ 视觉回归 | 最完整分层 |
| 前端组件库 | ① 单元 → ② 组件测试 → ③ 视觉回归 | 组件测试验证渲染、属性变化、事件触发 |

## 各层详细定义

### ① 单元测试（必选，所有项目）

- **测试对象**: 纯函数、工具、状态计算、校验规则、hooks
- **校验目标**: 给定入参 → 固定结果，锁代码逻辑规则
- **执行方式**: 机器化（Vitest/Jest）
- **命令**: `npm run test:unit`

### ② 集成测试（必选，所有项目）

- **测试对象**: 多单元组合的内部流程链路
- **校验目标**: 局部流程验收，不碰真实接口
- **执行方式**: 机器化
- **命令**: `npm run test:integration`

### ③ 组件测试（前端组件库）

- **测试对象**: 组件渲染、属性变化、事件触发
- **校验目标**: 介于单元和 E2E 之间
- **执行方式**: 机器化（Vitest + Testing Library）
- **命令**: `npm run test:component`

### ④ 接口契约测试（API 类项目）

- **测试对象**: 单个接口的格式、参数、返回结构
- **校验目标**: 单个接口本身合法
- **执行方式**: 机器化（OpenAPI 校验 / 契约测试脚本）
- **命令**: `npm run test:contract`

### ⑤ 接口编排测试（多接口联动场景）

- **测试对象**: 多接口调用时序与状态联动
- **校验目标**: 登录→创建→查询→编辑→删除 等串行流程
- **执行方式**: 机器化（串行接口脚本）
- **命令**: `npm run test:orchestration`

### ⑥ E2E 测试（有 UI 的项目）

- **测试对象**: 完整用户交互流程（页面、弹窗、路由、异步）
- **校验目标**: 跨接口、跨页面、状态流转
- **执行方式**: 机器化 + Agent 化（Playwright/Cypress + Agent 语义判断）
- **命令**: `npm run test:e2e`
- **注意事项**: 输出摘要化，只返回失败用例名/失败断言/截图路径

### ⑦ 视觉回归测试（前端项目）

- **测试对象**: 像素差异比对
- **校验目标**: 布局、样式、弹窗位置等肉眼问题
- **执行方式**: 机器化（Playwright + 像素比对）
- **命令**: `npm run test:visual`
- **注意事项**: 只输出差异百分比 + 差异区域路径，不嵌入图片
