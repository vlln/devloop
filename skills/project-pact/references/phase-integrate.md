# INTEGRATE 阶段

## 固定阶段

INTEGRATE 是唯一串行关口——必须等所有轨道 Plan done 才能进入。执行全量测试，禁止新增功能代码。

**INTEGRATE 不可跳过。** DEVELOP 的单元测试通过不代表可以通过 INTEGRATE。全量测试是独立于开发自测的验证阶段。

## 子阶段

### 全量测试（必选，所有项目）

按层级顺序执行，不可跳过。具体命令见 CONTRIBUTING.md 或项目测试脚本配置：

```
1. 单元测试
2. 集成测试
3. 接口契约测试（如果有）
4. 接口编排测试（如果有）
5. E2E 测试（如果有 UI）
6. 视觉回归测试（如果是前端）
```

每层通过后才进入下一层。

### 修复循环（适用：测试失败时）

每层失败时：
1. 停止，不继续下一层
2. 提取失败信息：失败用例名、失败断言、截图路径
3. 退回 DEVELOP 对应轨道修复
4. 修复完成后，重新从失败的那层开始

修复规则：
- 只修 bug，不新增功能
- 如果发现设计缺陷（不是 bug，是契约本身有问题）→ 更新 `docs/README.md` 当前阶段为 DESIGN，提交

## 推进到 PRE_RELEASE

全部测试通过后：更新 `docs/README.md` 当前阶段为 PRE_RELEASE，追加最近事件，提交。约定前缀 `docs(state):`。

## 输出格式

测试摘要（不要输出完整日志）：

```markdown
| 测试层 | 通过/总数 | 失败用例 | 耗时 |
|--------|----------|----------|------|
| 单元测试 | 12/12 | — | 2.3s |
| E2E | 2/3 | AC-007 (选择器超时) | 15.8s |

结论: ❌ 未通过 / ✅ 全部通过
```
